SOAL PENGAYAAN (Refleksi Polimorfisme)
Jawablah pertanyaan berikut:
1. Apa yang dimaksud dengan polimorfisme dalam OOP? Jelaskan bagaimana metode override bekerja.
2. Mengapa array bertipe DefaultMesin dapat menyimpan objek MesinMotor dan MesinTraktor?
3. Apa perbedaan override dan overload?
4. Apa keuntungan menggunakan polimorfisme dalam sistem inventaris seperti contoh di atas?
5. Bagaimana cara kerja dynamic dispatch pada Java saat memanggil method override?

JAWABAN
1. Polimorfisme dalam OOP adalah konsep di mana satu method yang sama dapat memiliki perilaku yang berbeda tergantung objek yang memanggilnya. Metode override bekerja ketika class anak mendefinisikan ulang method milik class induk dengan nama, parameter, dan tipe return yang sama, tetapi dengan isi yang berbeda. Saat method dipanggil melalui objek, Java akan menjalankan versi method milik class anak berdasarkan tipe objek sebenarnya (runtime), bukan berdasarkan tipe variabelnya.
2. karena keduanya merupakan subclass (turunan) dari defaultMesin. Dalam OOP, hal ini disebut polimorfisme, di mana objek dari class anak boleh disimpan dalam variabel bertipe class induk. Meskipun disimpan sebagai defaultMesin, saat method dipanggil yang dijalankan tetap method hasil override sesuai tipe objek aslinya.
3. Override terjadi ketika class anak menuliskan ulang method milik class induk dengan nama, parameter, dan return yang sama, tetapi isi method berbeda. Override berkaitan dengan inheritance dan terjadi saat runtime. Overload terjadi ketika beberapa method memiliki nama yang sama dalam satu class, tetapi parameter berbeda (jumlah atau tipe data berbeda). Overload tidak memerlukan inheritance dan ditentukan saat compile-time.
4. program menjadi lebih fleksibel dan efisien, karena semua jenis mesin (motor, traktor, traktor listrik) dapat diperlakukan sebagai satu tipe yang sama (defaultMesin) dalam satu array, tetapi tetap menjalankan perilaku yang berbeda sesuai jenis objeknya saat method dipanggil. Dengan polimorfisme, kode menjadi lebih sederhana, mudah dikembangkan, dan mudah dirawat, karena penambahan jenis mesin baru tidak memerlukan perubahan besar pada program utama.
5. dengan cara menentukan method yang dijalankan berdasarkan tipe objek sebenarnya saat program berjalan (runtime), bukan berdasarkan tipe variabelnya. Jadi meskipun sebuah objek disimpan dalam variabel bertipe parent, ketika method yang dioverride dipanggil, Java akan otomatis menjalankan method milik class anak yang sebenarnya, sehingga perilaku method dapat berbeda untuk setiap objek.
